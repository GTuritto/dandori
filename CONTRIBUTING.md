# Contributing to Dandori

Dandori is an open framework designed to evolve through practitioner feedback. Contributions are welcome in several forms.

## Experience Reports

The most valuable contributions are real-world experience reports. If you have adopted Dandori (fully or partially) with your team, open an issue describing what worked, what did not, and what you adapted. These reports directly inform framework improvements.

## Ceremony Refinements

If you have found that a ceremony works better with a different format, frequency, or Liberating Structure than what the framework recommends, open a pull request with your proposed change. Include the context: team size, domain, AI tools used, and what problem the change solved.

## Spec Templates

The framework defines what a specification should contain but deliberately does not prescribe a template. If you have developed spec templates that work well for your team, consider contributing them to the `templates/` directory.

## Corrections and Clarifications

If something in the framework is ambiguous, contradictory, or unclear, open an issue. Clarity is a core value of Dandori. If the framework itself is imprecise, it is not living up to its own principles.

## What We Are Not Looking For

- Theoretical arguments without practical experience
- Proposals that add ceremony without justifying the failure mode it addresses
- Changes that assume a specific AI tool or vendor

## Process

1. Open an issue describing the change and its rationale
2. Discuss with maintainers and community
3. Submit a pull request if the change is accepted
4. Include relevant context about your team and experience
